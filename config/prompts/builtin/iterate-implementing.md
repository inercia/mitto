Read the state file and relevant source files first. Do not speculate about
code you haven't opened.

Continue implementing features using a state file to track progress across iterations.

Only implement what's in the spec. No extra features or abstractions.

# Preparation

State file: `implement-<problem>-<date>.md` (date as `YYYY-MM-DD`).

Template:
```markdown
# The problem
[description]
# Progress
## [num] [issue title]
[description] [fix applied]
# Issues remaining
## [num] [issue title]
```

# The Task

1. Read state file (create if missing)
2. Review current state and what's still missing
3. Analyze remaining issues
4. Prioritize and implement next work item
5. Verify implementation
6. Update state file

Once complete, verify against original problem description.
